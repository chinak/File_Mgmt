## 一、项目概述

### 1.1 项目背景

本系统为金融机构设计的**合同档案管理系统**，用于管理客户开户合同从创建、审核、归档到完结的全生命周期。系统涉及分支机构、运营部门、综合部三方协作，需要跟踪纸质合同的物理流转和电子化归档过程。

### 1.2 项目目标

1. 实现合同档案的电子化管理，建立完整的档案台账
2. 规范纸质合同的寄送、审核、回寄、归档流程
3. 提供完整的状态追踪和操作审计日志
4. 支持 OCR 智能识别，提升数据录入效率
5. 实现基于角色的权限控制和数据隔离

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 档案记录 | 一份客户开户合同对应的系统记录 |
| 主流程 | 纸质合同从分支寄出到总部审核再回寄的物理流转过程 |
| 归档流程 | 审核通过的合同由运营转交综合部入库的内部流程 |
| 电子版合同 | 线上签署的电子合同，无需物理流转 |
| 纸质版合同 | 线下签署的纸质合同，需要完整的寄送和归档流程 |
| 资金账号 | 客户的唯一标识，系统内不可重复 |

## 四、功能需求

### 4.1 用户认证模块

#### FR-AUTH-001 用户登录

**功能描述**：用户通过用户名和密码登录系统

**输入**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| username | string | ✓ | 用户名 |
| password | string | ✓ | 密码 |

**处理逻辑**：
1. 校验用户名是否存在
2. 校验密码是否正确（bcrypt 哈希比对）
3. 生成 JWT Token（有效期 8 小时）
4. 返回 Token 和用户基本信息

**输出**：
| 字段 | 类型 | 说明 |
|------|------|------|
| token | string | JWT 访问令牌 |
| user.id | string | 用户 ID |
| user.username | string | 用户名 |
| user.role | string | 用户角色 |
| user.branchName | string | 所属营业部（分支机构用户） |

**异常处理**：
| 场景 | 错误码 | 提示信息 |
|------|--------|----------|
| 用户名不存在 | INVALID_CREDENTIALS | 用户名或密码错误 |
| 密码错误 | INVALID_CREDENTIALS | 用户名或密码错误 |

---

### 4.2 档案管理模块

#### FR-ARCHIVE-001 Excel 批量导入

**功能描述**：运营人员通过上传 Excel 文件批量导入档案记录

**前置条件**：用户角色为运营人员

**输入**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| file | File | ✓ | Excel 文件（.xlsx/.xls） |

**Excel 模板格式**：
| 列名 | 必填 | 说明 | 示例 |
|------|:----:|------|------|
| 客户姓名 | ✓ | 客户全名 | 张三 |
| 资金账号 | ✓ | 唯一标识 | 6001234567 |
| 营业部 | ✓ | 所属营业部名称 | 上海营业部 |
| 合同类型 | ✓ | 合同业务类型 | 股票开户 |
| 开户日期 | ✓ | 格式 YYYY-MM-DD | 2026-03-01 |
| 合同版本类型 | ✓ | 电子版/纸质版 | 纸质版 |

**处理逻辑**：
1. 校验文件格式（仅支持 .xlsx、.xls）
2. 逐行解析并校验：
   - 必填字段完整性
   - 合同版本类型值域（电子版/纸质版）
   - 资金账号文件内查重
   - 资金账号数据库查重
3. 根据合同版本类型设置初始状态：
   - 纸质版：`status=pending_shipment`, `archive_status=archive_not_started`
   - 电子版：`status=null`, `archive_status=archived`
4. 创建档案记录

**输出**：
| 字段 | 类型 | 说明 |
|------|------|------|
| totalRows | number | 总行数 |
| successCount | number | 成功导入数 |
| failureCount | number | 失败数 |
| errors | array | 错误详情列表 |
| errors[].row | number | 错误行号 |
| errors[].reason | string | 失败原因 |

### 4.3 状态流转模块

#### FR-TRANS-001 单条状态流转

**功能描述**：执行单条档案记录的状态变更操作

**输入**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| archiveId | string | ✓ | 档案 ID（URL 参数） |
| action | string | ✓ | 操作类型 |

**支持的操作类型**：

| action | 操作名称 | 执行角色 | 状态字段 | 前置状态 | 目标状态 |
|--------|----------|----------|----------|----------|----------|
| confirm_shipment | 确认寄出 | branch | status | pending_shipment | in_transit |
| confirm_received | 确认收到 | operator | status | in_transit | hq_received |
| review_pass | 审核通过 | operator | status | hq_received | review_passed |
| review_reject | 审核不通过 | operator | status | hq_received | review_rejected |
| return_branch | 发起回寄 | operator | status | review_passed/review_rejected | pending_return |
| confirm_shipped_back | 确认已寄出 | operator | status | pending_return | return_in_transit |
| confirm_return_received | 确认收到回寄 | branch | status | return_in_transit | branch_received |
| transfer_general | 转交综合部 | operator | archive_status | pending_transfer | pending_archive |
| confirm_archive | 确认入库 | general_affairs | archive_status | pending_archive | archived |

**联动逻辑**：

1. **review_pass 联动**：
   - 主变更：`status: hq_received → review_passed`
   - 联动变更：`archive_status: archive_not_started → pending_transfer`

2. **confirm_return_received 自动判断**：
   - 主变更：`status: return_in_transit → branch_received`
   - 若 `archive_status = archive_not_started`：自动回退 `status → pending_shipment`
   - 若 `archive_status = archived`：自动完结 `status → completed`

**校验规则**：
1. 电子版合同拒绝所有状态变更
2. 已完结记录拒绝所有状态变更
3. 操作人角色必须匹配
4. 当前状态必须允许目标转换

---

```
│                                         │ archive      │                   │
│                                         └──────────────┘                   │
│                                                │                            │
│                                                │ confirm_archive            │
│                                                │ (综合部)                    │
│                                                ▼                            │
│                                         ┌──────────────┐                   │
│                                         │   已归档      │                   │
│                                         │  archived    │                   │
│                                         └──────────────┘                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.3 典型业务场景

#### 场景1：纸质合同正常流程（审核通过）

```
1. 运营人员导入档案数据 → status=pending_shipment, archive_status=archive_not_started
2. 分支机构确认寄出 → status=in_transit
3. 运营人员确认收到 → status=hq_received
4. 运营人员审核通过 → status=review_passed, archive_status=pending_transfer (联动)
5. 运营人员发起回寄 → status=pending_return
6. 运营人员确认已寄出 → status=return_in_transit
7. 运营人员转交综合部 → archive_status=pending_archive
8. 综合部确认入库 → archive_status=archived
9. 分支机构确认收到回寄 → status=completed (自动完结)
```

#### 场景2：纸质合同审核不通过流程

```
1. 运营人员导入档案数据 → status=pending_shipment
2. 分支机构确认寄出 → status=in_transit
3. 运营人员确认收到 → status=hq_received
4. 运营人员审核不通过 → status=review_rejected (archive_status 保持 archive_not_started)
5. 运营人员发起回寄 → status=pending_return
6. 运营人员确认已寄出 → status=return_in_transit
7. 分支机构确认收到回寄 → status=pending_shipment (自动回退，重新开始流程)
```

#### 场景3：电子版合同

```
1. 运营人员导入/新建档案 → status=null, archive_status=archived (创建即完结)
2. 无后续状态流转操作
```

| 类别 | 要求 |
|------|------|
